Redis Distributed Lock — Проверка работы в Cron-сервисе

Этот README описывает, как работает механизм распределённого лока в Redis, зачем он нужен, и как протестировать его работу при запуске нескольких инстансов cron-сервиса.

Зачем нужен Redis-лок

Когда несколько инстансов сервиса запускают один и тот же периодический процесс (например, генерацию рекомендаций), важно избежать ситуации, когда все инстансы начинают выполнение одновременно.

Для этого используется распределённый лок на Redis. Только один инстанс сможет захватить лок — остальные получат ошибку вида:

redis lock: already acquired

и не запустят тяжёлую задачу.


⚙️ Как работает лок
	1.	Каждый инстанс пытается выполнить команду SET с параметрами NX и EX, например:

SET lock_key value NX EX 60

	2.	Условие NX означает: создать ключ, только если он ещё не существует.
	3.	Если ключ уже существует (лок удерживает другой инстанс):
	•	Redis вернёт nil.
	•	В коде будет залогировано:
“redis lock already acquired”
	4.	Если ключ успешно создан:
	•	Инстанс выполняет крон-задачу.
	•	По окончании — удаляет ключ (освобождает лок).

⸻

Как протестировать работу лока

1. Подготовка

Перед тестом остановите работающий cron-контейнер:

docker stop cron-service

2. Очистите Redis (только для локальной разработки!)

❗ Внимание: это удалит все данные Redis.

redis-cli FLUSHALL

Проверьте, что ключей нет:

redis-cli KEYS '*'

Ожидаемый результат: пустой вывод.

⸻

3. Запустите несколько инстансов одновременно

Если используете docker-compose:

docker-compose up --scale cron-service=3 --build

Или в окружении Kubernetes:

kubectl scale deployment cron-service --replicas=3


⸻

4. Проверьте логи

Откройте логи всех контейнеров/подов:

docker-compose logs -f cron-service

Ожидаемая картина:
	•	Один инстанс пишет что-то вроде:

redis lock acquired, starting job


	•	Остальные показывают:

redis lock already acquired



Если хотя бы один контейнер выводит "redis lock already acquired", значит лок работает правильно.

⸻

Полезные команды

Посмотреть ключи лока

redis-cli KEYS 'lock*'

Проверить TTL лока

redis-cli TTL lock_key

Удалить только ключ лока

redis-cli DEL lock_key


⸻

Ожидаемый результат проверки
	•	Только один из инстансов запускает крон-задачу.
	•	Остальные выводят лог "redis lock already acquired".
	•	После завершения работы первого инстанса ключ в Redis исчезает или истекает по TTL.
